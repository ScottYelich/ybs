## Notes

### Decision Rationale

**Why centralize scripts?**
- Scripts should not be buried in documentation directories
- Follows Unix convention: executables in `bin/`
- Easier to discover and run
- Clear separation: docs = documentation, bin = executables

**Why rename yds → ybs?**
- "Derp" was unprofessional for a public repository
- "Build System" accurately describes purpose (framework for building LLM assistants)
- Consistency: YBS = Yelich Build System
- Better branding for potential wider adoption

**Why session-based changelogs?**
- Supports parallel work (multiple contributors or AI agents)
- No merge conflicts on single CHANGELOG.md
- Clear session boundaries and scoping
- Easier to find changes by date or session
- Scales to many contributors
- Each session is self-contained with full context
